# Onboarding — Cursor & agentic AI

**Audience:** New users learning **Cursor**, **Plan vs Agent** workflows, and **human–agent collaboration** (especially PM-style work).

**Purpose:** Central place for **shareable** artifacts — guides, checklists, snippets, and curated links — without mixing them into `.cursor/` (which is machine-oriented rules/skills).

## How to use this folder

| Path | Contents |
|------|----------|
| [`guides/`](guides/) | Longer tutorials and playbooks |
| [`checklists/`](checklists/) | Short first-week / pre-share lists |
| [`snippets/`](snippets/) | Copy-paste prompt or handoff templates |
| [`links/`](links/) | External references (Cursor docs, security, ethics) |
| [`INDEX.md`](INDEX.md) | What exists, draft vs ready, publication status |

## Relationship to the rest of the repo

- **`.cursor/skills/`** — Procedures the *agent* loads; skills may **point here** for human-readable depth.
- **`cursor-knowledge/`** — Session knowledge log and philosophy (e.g. [`BACKLOG_PHILOSOPHY.md`](../cursor-knowledge/BACKLOG_PHILOSOPHY.md)).
- **[`BACKLOG.md`](../../BACKLOG.md)** (repo root) — **Prioritized** workspace backlog (see [backlog-prioritization](../../.cursor/rules/backlog-prioritization.mdc) rule).

## Before publishing externally

1. Remove or redact secrets, tokens, internal paths if uncomfortable.
2. Prefer linking to [`vcrosby22/cursor-knowledge`](https://github.com/vcrosby22/cursor-knowledge) for public clones rather than dumping private `journal/`.
