# First week with Cursor (non-technical)

Use this as a **light checklist**. Skip steps that do not apply; add your own goals in the notes row.

## Days 1–2: get oriented

- [ ] Install **Cursor**, sign in, and open **one small folder** (even a blank folder with a single `notes.md` is enough).
- [ ] Open **chat** and try **three** different goals (pick your own wording):
  - [ ] **Outline** — e.g. “Turn these bullets into a one-page outline with clear sections.”
  - [ ] **Critique** — e.g. “What are the main risks and gaps in this idea?”
  - [ ] **Draft** — e.g. “Draft a short paragraph I can use on a landing page; I will edit tone.”
- [ ] Read **[Building with agentic AI (non-technical)](../guides/building-with-agentic-ai-non-technical.md)** — at least sections 1–4 (mindset, safety, Plan vs Agent, first wins).

## Days 3–4: modes and context

- [ ] Run **one** task in **Plan** mode (think together, minimal or no file changes) and the **same kind of task** in **Agent** mode (let the assistant update files if you are comfortable). Notice the difference.
- [ ] Read **[Working with AI context](../guides/working-with-ai-context.md)** — especially when to start a **new chat**.
- [ ] Start a **new chat** once on purpose with a **short handoff** (goal + what not to assume) — see the template in that guide.

## Days 5–7: optional “project” bridge (still light on terminal)

- [ ] Skim the **[session playbook](../guides/cursor-session-playbook.md)** intro; bookmark it for when you add Git or a static site.
- [ ] If you need **version control** or **GitHub**, work through **[Git basics for non-developers](../guides/git-basics-for-non-developers.md)** at your own pace.
- [ ] Browse **[curated links](../links/README.md)** and open two references you will actually use (docs, security, or verification).
